Hello!

The following is a tour throughout the different components and styles on Cycledash. We use Bootstrap, which means even more documentation can be found on their website. Happy styling!

Buttons

Buttons are used to perform an action. For example, when submitting a form or examining data.

Primary action

Use these to direct your user to the most important actions on the page.

Secondary action

These acknowledge that there is an action to take, but that it's not the most important one.

Form submit

These are to denote that the action will submit or save data.

+ Glyphicon

Adding a universal symbol can make the action clearer and more understandable.

Cards

Turning information into something physical adds hierarchy and organization to the page.

Icons

Icons add a visual language that will grab your user's attention and increase their understanding of the actions and elements on the page. Icons should be used sparingly, for too many can result in dilution and decoration. All icons in Cycledash are using Bootstrap glyphicons.

Modals

Use modals when you want your user to focus on performing a particular task. For example, uploading data or reading an important disclaimer.

Tables

Tables are used to collect and show groups of tabular data. For example, projects and vcfs are arranged in tables with headers that apply to each column of data. Some rows expand to reveal more information about that particular row.

Typography

Different font sizes, colors and emphases help convey hierarchy of information on a page. Consistency is key to building patterns that will help your user recognize and parse information effortlessly.

  • Hello, I'm an h1

  • Hello, I'm an h2

  • Hello, I'm an h3

  • Hello, I'm an h4

  • Here is a link
  • This is deemphasized text, commonly used in empty states and time stamps.
